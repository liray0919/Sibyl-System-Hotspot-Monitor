# 版本更新记录

[English](CHANGELOG.en.md) | [中文 README](README.zh-CN.md)

## 1.2.5 - 2026-06-22

- 新增 `scripts/check-deploy.js` 部署后自检脚本。
- 自检覆盖 webhook、微博智搜、AI 模型、系统时区、数据目录和人工确认项。
- 自检输出只显示“已配置/未配置”，避免打印真实 webhook、AppSecret 或模型密钥。

## 1.2.4 - 2026-06-22

- 微博智搜凭据和 endpoint 支持环境变量配置。
- 新增 `alerts.weiboAI` 配置项，允许在 Sibyl 配置文件中设置微博智搜 appId、appSecret、authUrl 和 searchUrl。
- 保留 `/root/.openclaw/openclaw.json` 中 `channels.weibo` 的原有配置作为回退。

## 1.2.3 - 2026-06-22

- 将综合分展示档位简化为三档：`观察`、`关注后续`、`强热点`。
- 明确综合分是排序指标，不是跨平台绝对热度总和。
- 以 `Sibyl System Hotspot Monitor` 为项目名完成 GitHub 发布准备。

## 1.2.2 - 2026-06-22

- 补充定时任务中显式注入 Webhook 环境变量的部署说明。
- 增加系统时区一致性检查说明，避免北京时间调度窗口被错误解释。
- 明确 OpenClaw 任务投递模式，避免重复推送。

## 1.2.1 - 2026-06-18

- 新增 Sibyl System 内部产品文档。
- 补充产品目标、用户场景、架构图、数据流、部署、复用指南和风险边界。
- 增加面向产品和运营评审的分享讲法。

## 1.2.0 - 2026-06-18

- 新增即时预警和小时综合播报的独立 Webhook 配置。
- 完善即时预警摘要链路：缓存、旧摘要复用、微博智搜、Elixir 摘要、AI 搜索兜底。
- 增加商业类和非目标内容过滤。
- 为小时综合播报增加 AI 处理建议，并保留规则兜底。

## 1.1.0 - 2026-06-18

- 升级为统一快照架构。
- 即时预警和小时综合播报共用 10 分钟快照。
- 将微博、抖音即时预警整合进 Sibyl 数据流。
- 增加近 70 分钟快照趋势判断。
- 新增贴吧、虎扑作为社区补充信号。

## 1.0.0 - 2026-06-16

- 初始版本。
- 支持微博、抖音、贴吧、虎扑多平台信号采集。
- 增加话题分类、事件聚合、综合评分和 Markdown 报告生成。
- 支持企业微信综合播报推送。
